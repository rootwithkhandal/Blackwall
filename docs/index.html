Architecture & Blockchain Integrity

The BlackWall architecture separates concerns. Background daemon threads capture traffic, the ML detector evaluates it, and verdicts are dispatched both to system-level firewalls and external SIEMs simultaneously.

Frequently Asked Questions

Everything you need to know about BlackWall.

What is BlackWall? +
BlackWall is an open-source, ML-powered Security Operations Center and firewall engine that brings enterprise-level network monitoring to your local machine.
How does the ML anomaly detection work? +
It uses an Isolation Forest algorithm from scikit-learn. The system profiles your baseline traffic for the first 5 minutes. After that, any sudden spikes in packet rates or unusual port activity are flagged as anomalies and automatically blocked.
Why does it use a Blockchain ledger? +
A key requirement for incident response and compliance is untampered logs. BlackWall chains every firewall rule and drop verdict into a cryptographic ledger. If a malicious actor gains access and tries to delete their logs, the integrity check will immediately flag the tampering.
Does it run on Windows or Linux? +
It runs on both! On Linux, BlackWall seamlessly integrates with iptables to enforce bans at the kernel level. On Windows, it gracefully falls back to application-layer packet inspection without breaking the dashboard.
